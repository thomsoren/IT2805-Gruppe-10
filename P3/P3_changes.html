Changes that we made

individualWatches.html

I (henrik) worked with individualWatches.html. The descriptions that were made in p2 were mostly visual, and as such I have taken a few liberties when it comes to function. We have several "products" and as such i wanted to consolidate all the different products into only individualWatches.html. This was achived through just adding a line to the url (in our case product1-6 and make the website display different pictures, prices, names and descriptions accordingly.) as for the styling, i did not follow the size requierments that were specified for the images because i thought it was too big and looked ugly. Also i made the link change depending on the url so that the price in checkout can be changed by reusing the same code.

universal.js

I (henrik) just made the navbar and footer into a variable and innerHTMLed it to all the websites.

contact.html

The main changes to this website is in the information in the box off to the right of the form. The information is the same, but the styling of the information is not. "Contact" has been pushed beneath "Watch me Trade" instead of to the right of it

The submit button has also not been styled the exact same way as in the mockup, but it is mostly identical

For the mobile version, I decided to push the information down beneath the form instead of to the right, and have the user scroll down for the information instead.

error.html

This has just been styled to be more appealing to look at. We made the message into two sentences to seem more apologetic

Changes to buy.html

The mayor differece is that pictures, title and price got added to each box on the site. The header "Browse our watches" got added because it makes it easier for the user to understand where on the site the used is located. A "view more" button got added because it makes the boxes more intuivite to click. For the "view more"-button to fit and look good, we had to move the title, prince and button centered underneath eachother, instead of side by side.

Changes in universal layout

I (Ludvig) made a few visual changes to the header, like removing the underline, and changing all the font sizes to vw to be more fluid. Becoming bold, and changing color didn't look very good either, so instead, I made a function in javascript called applyScaleEffect. This function scales all sitelinks to 10% of it's original style. I also applied a transition effect to make it look more smoothly.

When the window size goes to a ipad window size or smaller, the horisontal page links changes to a button, that when hovered, will show the page links as a dropdown menu. Used @media (max-width: 768px) for this, and we used this media querie as the default when styling all the other pages as well.

We were supposed to have the current page you were visiting scale by 10%, but we removed this feature as it didn't feel as responsive as we thought it would be.

Changes to frontpage.html

The frontpage looks pretty much the same as it was supposed to, except from making the div boxes that each watch is to be displayed wider. I also made a media querie so that each box is displayed on top of each other, with the same margin on all sides.

sell.html

There are no huge changes from the plan when it came to this website as the plan for how it would look and what types the input should be worked mostly without any complications. The group hadn't specified it but there had to be a small change with uploading a picture. The contents of the image input would remain hidden until the user uploaded a valid image where it with javascript would set the style back to visible so the user can see the file uploaded. The default button which says "upload file" is also removed as it does not fit our design.
